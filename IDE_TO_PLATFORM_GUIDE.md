# 🔧 解决：IDE项目在Coze平台看不到的问题

## 😊 问题原因

**Coze Coding IDE** 和 **Coze平台** 是两个独立的环境：
- IDE = 开发环境（写代码的地方）
- Coze平台 = 发布环境（用户使用的地方）

您的项目目前只在IDE中，还没有发布到Coze平台，所以在平台上看不到。

---

## 💡 解决方案：两种方式任选

### 方案一：在Coze平台手动创建Bot（最快，推荐！）

**优点：** 5分钟完成，无需同步，立即获得分享链接

#### 步骤：

**1. 打开Coze平台**
```
https://www.coze.cn
```

**2. 点击右上角「创建Bot」**

**3. 填写基本信息**
- **Bot名称**：简历战斗力分析
- **Bot简介**：专业简历分析助手，5分钟深度诊断简历问题

**4. 配置系统提示词**（关键步骤）

点击左侧「编排」→「人设与回复逻辑」，复制以下内容：

```

# 任务目标
对用户提供的简历进行深度分析，输出一份专业的战斗力评估报告。

# 能力
- 简历结构化分析：提取关键信息，识别缺失模块
- 竞争力评分：从技能匹配度、业务理解深度、项目质量、表达清晰度四个维度量化评分
- 问题诊断：识别简历中的致命问题，解释为什么会被HR筛掉
- 优化改写：提供改前VS改后的对比示例，给出具体修改建议
- 行动建议：给出高、中、低优先级的改进行动清单

# 分析流程
1. 接收简历内容，判断完整性
2. 提取核心信息（工作经历、项目、技能、教育背景）
3. 根据目标岗位要求，进行四维评分
4. 识别三大致命问题
5. 针对每个问题提供改前VS改后的优化示例
6. 输出结构化的战斗力分析报告
7. 询问用户是否需要PDF版本
```

### 方案二：同步IDE项目到Coze平台（复杂，需要Git）

**如果您需要保留IDE中的所有功能（如PDF生成），需要同步代码：**

#### 步骤：

**1. 在IDE中推送代码到Git仓库**

在IDE终端执行：
```bash
cd /workspace/projects

# 初始化Git（如果还没有）
git init
git add .
git commit -m "初始化简历分析Agent"

# 推送到GitHub/Gitee
git remote add origin https://github.com/你的用户名/resume-analyzer.git
git push -u origin main
```

**2. 在Coze平台导入项目**

1. 访问：https://www.coze.cn
2. 点击「创建项目」
3. 选择「从Git导入」
4. 输入您的仓库地址
5. 配置环境变量
6. 构建和部署
